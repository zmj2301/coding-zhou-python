# 口算战争项目文档

## 一、项目概述

**口算战争**是一款面向儿童的益智教育游戏，将数学口算练习与游戏玩法相结合。游戏采用塔防射击的游戏机制，玩家需要正确回答数学问题来发射炮弹击落下落的陨石，保护基地不被摧毁。

### 1.1 项目定位
- **目标用户**：小学生及学龄前儿童
- **核心目标**：通过游戏化方式提升儿童口算能力
- **游戏类型**：教育益智类塔防射击游戏

### 1.2 主要功能特性
- 🎮 双游戏模式：口算战争模式 + 飞机大战模式
- 📚 关卡系统：10个递进难度关卡
- 🧮 口算练习：加法运算题目生成与验证
- 🎯 射击机制：炮台瞄准与射击系统
- 🛡️ 奖励系统：连续答对获得奖励道具
- ⏸️ 暂停功能：支持游戏暂停/继续

#### 3.1.2 核心功能函数

| 函数名 | 功能描述 | 关键逻辑 |
|--------|----------|----------|
| `load_image()` | 加载游戏资源 | 加载所有图片、音效资源到全局变量 |

| `run_rock_bullet()` | 游戏主循环 | 处理陨石移动、碰撞检测、炮台旋转 |
| `play_oral()` | 游戏逻辑控制 | 处理用户输入、答案验证、状态更新 |

#### 3.1.4 核心游戏机制

**1. 口算题目生成**
```python
def set_question():
    n3 = '+'  # 当前仅支持加法
    n1 = random.randint(1,100)
    n2 = random.randint(1,100)
    question = str(n1) + str(n3) + str(n2) + '='
    answer = n1 + n2
```

**2. 碰撞检测系统**
- 子弹与陨石碰撞：使用 `pygame.Rect.colliderect()`
- 陨石与红线碰撞：检测游戏结束条件
- 子弹与护盾碰撞：增加生命值

**3. 炮台瞄准系统**
```python
# 炮台自动追踪最近的陨石
crosshair_x, crosshair_y = red_crosshair_pos
dx = crosshair_x - 390
dy = crosshair_y - 690
battery_angle = math.degrees(math.atan2(-dy, dx))
rotated_battery = pygame.transform.rotate(battery_1, battery_angle-90)
```

**4. 奖励机制**
- 连续答对指定题目数量触发奖励模式
- 奖励模式：玩家控制飞机发射子弹，陨石停止移动
- 奖励结束后恢复正常游戏

## 四、游戏流程详解

### 4.1 口算战争模式流程

```
┌─────────────────────────────────────────────────────────────┐
│                        游戏启动                              │
└──────────────────────┬──────────────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────────────┐
│                     开始界面 (content=0)                     │
│              显示"开始游戏"按钮                               │
└──────────────────────┬──────────────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────────────┐
│                    模式选择 (content=1)                      │
│         口算战争练习  |  无线模式(飞机大战)                    │
└──────────────────────┬──────────────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────────────┐
│                    关卡选择 (content=2)                      │
│              显示10个关卡，带锁定状态                          │
└──────────────────────┬──────────────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────────────┐
│                    游戏进行 (content=3)                      │
│  1. 显示口算题目                                             │
│  2. 玩家输入答案（数字键盘/键盘输入）                          │
│  3. 答案正确→发射炮弹→击碎陨石→得分                          │
│  4. 答案错误→显示红色提示                                     │
│  5. 陨石到达红线→扣减生命值                                   │
│  6. 生命值归零→游戏结束                                       │
└──────────────────────┬──────────────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────────────┐
│                      关卡完成                                │
│              解锁下一关，返回关卡选择                          │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 核心交互逻辑

**答题流程：**
1. 玩家点击数字按钮或按键盘数字键输入答案
2. 点击"提交"按钮或按回车确认
3. 系统验证答案：
   - 正确：炮台发射，击碎陨石，生成新题目
   - 错误：输入框变红，等待重新输入

**暂停功能：**
- 点击暂停按钮或按空格键
- 游戏画面冻结，音乐暂停
- 再次点击恢复游戏

## 六、开发流程与规范

### 6.1 开发环境搭建

```bash
# 1. 创建虚拟环境
python -m venv .venv

# 2. 激活虚拟环境
.venv\Scripts\activate

# 3. 安装依赖
pip install pygame
```

### 6.2 代码规范

- **命名规范**：
  - 函数名：小写+下划线，如 `draw_to_oral_arithmetic`
  - 全局变量：小写，如 `running`, `score`
  - 常量：全大写，如 `FPS`, `WIDTH`

- **注释规范**：
  - 关键逻辑添加中文注释
  - 复杂算法说明实现思路

- **资源路径**：
  - 使用 `os.path.join()` 或 f-string 构建路径
  - 支持相对路径运行

### 6.3 调试技巧

1. **打印调试**：使用 `print()` 输出关键变量
2. **FPS显示**：通过 `clock.tick(FPS)` 控制帧率
3. **碰撞框显示**：取消注释 `pygame.draw.circle()` 查看碰撞范围

## 七、优化建议

### 7.1 代码结构优化

1. **模块化重构**
   - 将游戏拆分为多个模块：
     ```
     game/
     ├── __init__.py
     ├── main.py           # 程序入口
     ├── config.py         # 配置常量
     ├── sprites.py        # 精灵类定义
     ├── ui.py             # 界面绘制
     ├── question.py       # 题目生成
     └── utils.py          # 工具函数
     ```

2. **状态机模式**
   - 使用状态机管理游戏流程
   - 替代当前的 `content` 变量判断

3. **类封装**
   - 将全局变量封装到游戏类中
   - 使用面向对象方式组织代码

### 7.2 功能扩展建议

1. **题目类型扩展**
   - 添加减法、乘法、除法运算
   - 根据关卡难度调整数字范围
   - 添加混合运算题目

2. **游戏机制增强**
   - 陨石类型多样化（速度不同、血量不同）
   - 添加BOSS关卡
   - 连击系统与Combo奖励

3. **数据持久化**
   - 保存玩家进度
   - 记录最高分数
   - 统计答题正确率

4. **UI/UX改进**
   - 添加开始菜单动画
   - 优化数字键盘布局
   - 添加音效开关设置

### 7.3 性能优化

1. **资源加载优化**
   - 使用延迟加载（Lazy Loading）
   - 图片资源预加载到内存

2. **渲染优化**
   - 使用 `convert_alpha()` 优化图片
   - 脏矩形更新减少重绘区域

3. **内存管理**
   - 及时清理不再使用的对象
   - 使用对象池管理频繁创建销毁的对象

## 八、常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 图片加载失败 | 路径错误 | 使用 `os.path.dirname(os.path.abspath(__file__))` 获取绝对路径 |
| 字体显示异常 | 字体文件缺失 | 确保 `font.ttf` 存在于项目根目录 |
| 音效无法播放 | 音频格式不支持 | 使用 WAV 或 OGG 格式 |
| 游戏卡顿 | 帧率控制不当 | 确保调用 `clock.tick(FPS)` |
| 陨石重叠 | 位置检测逻辑问题 | 增加 `min_distance` 检测范围 |
