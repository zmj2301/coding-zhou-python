# 2026跨年庆祝程序

一个功能完整、视觉吸引力强的Python跨年庆祝程序，包含倒计时显示、烟花动画、主题定制等多种功能。

## 功能特性

### 1. 倒计时显示模块

- 实时精确计算并显示距离新年（1月1日00:00:00）的剩余时间
- 格式为"X天 X时 X分 X秒"
- 毫秒级时间更新机制，确保显示精准无误差
- 醒目的数字显示样式，采用节日主题配色方案
- 支持响应式布局，适配不同设备尺寸

### 2. 新年庆祝动画系统

- 新年零点自动触发的庆祝动画
- 包含烟花、彩带、闪烁灯光等视觉效果
- 动画性能优化，确保流畅运行（帧率不低于30fps）
- 提供手动触发动画的备用机制
- 支持动画重播功能

### 3. 主题定制功能

- 5种不同风格的倒计时背景主题：
  - 默认主题（深蓝紫色调）
  - 雪景主题（浅蓝色调）
  - 烟花主题（深色背景）
  - 中国红主题（红色调）
  - 星空主题（深蓝色调）
- 主题预览功能，允许实时切换
- 自动保存用户主题偏好设置

### 4. 新年愿望功能

- 用户友好的愿望输入界面
- 支持多行文本输入
- 愿望保存功能，使用本地JSON文件持久化用户数据
- 新年零点愿望展示特效

### 5. 节日氛围增强元素

- 集成音频管理系统（支持背景音乐播放/暂停和音量调节）
- 节日元素交互效果（雪花飘落等）
- 实时节日倒计时提醒

## 技术实现

### 开发环境

- Python 3.7+
- Pygame 2.5.2

```
趣味小游戏/
├── 2026跨年.py        # 主程序文件
├── config.json        # 配置文件（自动生成）
├── wishes.json        # 愿望存储文件（自动生成）
├── assets/            # 资源目录
│   ├── images/        # 图片资源
│   └── audio/         # 音频资源
└── README.md          # 项目说明文档
```

### 核心类设计

#### CountdownApp 主程序类

- 负责整体程序的初始化和运行
- 管理所有功能模块的协调工作
- 处理用户输入和事件响应

#### Firework 烟花类

- 实现烟花的发射、爆炸和粒子效果
- 包含烟花的物理运动和生命周期管理

#### Particle 粒子类

- 实现烟花爆炸后的粒子效果
- 包含粒子的颜色、速度、重力和生命周期

#### Ribbon 彩带类

- 实现飘落的彩带效果
- 包含彩带的颜色、长度和运动轨迹

#### Snowflake 雪花类

- 实现飘落的雪花效果
- 包含雪花的大小、速度和风力影响

### 配置文件说明

- `config.json`：存储用户的主题偏好、音量设置等
- `wishes.json`：存储用户的新年愿望

## 使用说明

### 运行程序

```bash
python 2026跨年.py
```

### 操作指南

#### 基本操作

- **主题切换**：点击左上角的"主题"按钮，选择喜欢的主题
- **播放动画**：点击"播放动画"按钮，手动触发庆祝动画
- **音乐控制**：点击"音乐"按钮切换音乐播放状态
- **音量调节**：拖动音量滑块调整音量大小
- **许下愿望**：点击"许下愿望"按钮，输入并保存新年愿望

#### 键盘快捷键

- **ESC**：关闭当前打开的面板（主题选择或愿望输入）
- **Enter**：在愿望输入框中按下Enter键保存愿望
- **Backspace**：在愿望输入框中删除字符

### 自定义功能

#### 添加自定义背景

1. 将图片文件放入 `assets/images` 目录
2. 在代码中修改 `custom_background` 配置项
3. 重启程序即可生效

#### 添加背景音乐

1. 将音频文件放入 `assets/audio` 目录
2. 在 `load_background_music` 方法中加载音频文件
3. 重启程序即可生效

## 性能优化

1. **限制动画元素数量**：控制烟花、彩带等元素的最大数量
2. **条件渲染**：仅在特定主题下渲染对应元素（如仅在雪景主题下渲染雪花）
3. **对象池复用**：避免频繁创建和销毁对象
4. **字体渲染缓存**：缓存已渲染的文本，减少重复渲染开销

## 开发说明

### 模块化设计

程序采用模块化设计，各功能模块相对独立，便于维护和扩展：

- 倒计时模块：负责时间计算和显示
- 动画模块：负责烟花、彩带等动画效果
- 主题模块：负责主题切换和管理
- 愿望模块：负责愿望的输入、保存和显示
- 音频模块：负责音频的播放和管理

### 代码风格

- 遵循PEP 8代码规范
- 类名采用大驼峰命名法
- 方法和变量名采用小驼峰命名法
- 代码注释清晰，便于理解
