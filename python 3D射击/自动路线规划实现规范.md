# 敌人自动路线规划实现规范

## 1. 项目背景

当前3D射击游戏中的敌人只能直线追击玩家，遇到墙壁会被阻挡，但不会绕路。为了提高游戏的可玩性和AI的智能程度，需要实现敌人的自动路线规划功能，使敌人能够绕过障碍物寻找到达玩家的最佳路径。

## 2. 需求分析

### 2.1 功能需求
- 敌人能够在迷宫中自动寻路到玩家位置
- 能够绕过墙壁等障碍物
- 路径计算高效，不影响游戏性能
- 支持动态路径更新（当玩家移动时）

### 2.2 技术需求
- 基于现有迷宫数据实现
- 兼容Ursina引擎
- 算法效率高，时间复杂度合理
- 代码结构清晰，易于维护

## 4. 技术方案

### 4.1 路径规划算法选择

**A*算法** - 推荐使用，理由：
- 高效的启发式搜索算法
- 能找到最优路径
- 适合网格类迷宫寻路
- 时间复杂度可控

### 4.2 算法实现原理

A*算法核心公式：
```
f(n) = g(n) + h(n)
```
- `f(n)`：节点n的总代价
- `g(n)`：从起点到节点n的实际代价
- `h(n)`：从节点n到终点的估计代价（启发函数）

### 4.3 迷宫网格转换

#### 4.3.1 坐标系统映射
- 游戏世界坐标 → 迷宫网格坐标
- 转换公式：
  ```python
  grid_x = int((world_x + 16) / 10)
  grid_z = int((world_z + 16) / 10)
  ```
- 10是墙壁的scale大小
- -16是迷宫的起始偏移

#### 4.3.2 可行走区域判断
- `maze_data[grid_x][grid_z] == 0` → 可行走
- `maze_data[grid_x][grid_z] == 1` → 障碍物

### 4.4 路径规划实现架构

## 5. 详细实现设计

### 5.1 GridNode类设计

```python
class GridNode:
    def __init__(self, x, z, walkable=True):
        self.x = x              # 网格X坐标
        self.z = z              # 网格Z坐标
        self.world_x = x * 10 - 16  # 转换为世界X坐标
        self.world_z = z * 10 - 16  # 转换为世界Z坐标
        self.walkable = walkable   # 是否可行走
        
        # A*算法相关属性
        self.g = float('inf')    # 到起点的实际代价
        self.h = 0               # 到终点的估计代价
        self.f = float('inf')    # 总代价
        self.parent = None       # 父节点
    
    def __lt__(self, other):
        return self.f < other.f
```

### 5.2 AStarPathfinder类设计

```python
class AStarPathfinder:
    def __init__(self, maze_data):
        self.maze_data = maze_data
        self.grid = self._create_grid()
    
    def _create_grid(self):
        """根据迷宫数据创建网格"""
        grid = []
        for x in range(len(self.maze_data)):
            row = []
            for z in range(len(self.maze_data[0])):
                walkable = self.maze_data[x][z] == 0
                node = GridNode(x, z, walkable)
                row.append(node)
            grid.append(row)
        return grid
    
    def _get_neighbors(self, node):
        """获取相邻节点（上、下、左、右）"""
        neighbors = []
        directions = [(-1, 0), (1, 0), (0, -1), (0, 1)]
        
        for dx, dz in directions:
            new_x = node.x + dx
            new_z = node.z + dz
            
            if 0 <= new_x < len(self.grid) and 0 <= new_z < len(self.grid[0]):
                neighbor = self.grid[new_x][new_z]
                if neighbor.walkable:
                    neighbors.append(neighbor)
        
        return neighbors
    
    def _calculate_h(self, node, end_node):
        """使用曼哈顿距离作为启发函数"""
        return abs(node.x - end_node.x) + abs(node.z - end_node.z)
    
    def find_path(self, start_world_pos, end_world_pos):
        """寻找从起点到终点的最佳路径"""
        # 转换世界坐标到网格坐标
        start_x = int((start_world_pos[0] + 16) / 10)
        start_z = int((start_world_pos[2] + 16) / 10)
        end_x = int((end_world_pos[0] + 16) / 10)
        end_z = int((end_world_pos[2] + 16) / 10)
        
        # 边界检查
        if (start_x < 0 or start_x >= len(self.grid) or 
            start_z < 0 or start_z >= len(self.grid[0]) or
            end_x < 0 or end_x >= len(self.grid) or
            end_z < 0 or end_z >= len(self.grid[0])):
            return []
        
        start_node = self.grid[start_x][start_z]
        end_node = self.grid[end_x][end_z]
        
        # 初始化开放列表和关闭列表
        open_list = []
        closed_list = set()
        
        # 添加起点到开放列表
        start_node.g = 0
        start_node.h = self._calculate_h(start_node, end_node)
        start_node.f = start_node.g + start_node.h
        heapq.heappush(open_list, start_node)
        
        while open_list:
            # 获取f值最小的节点
            current_node = heapq.heappop(open_list)
            
            # 到达终点，回溯路径
            if current_node == end_node:
                return self._reconstruct_path(current_node)
            
            closed_list.add(current_node)
            
            # 检查所有邻居
            for neighbor in self._get_neighbors(current_node):
                if neighbor in closed_list:
                    continue
                
                # 计算到邻居的临时g值
                temp_g = current_node.g + 1  # 相邻节点距离为1
                
                if temp_g < neighbor.g:
                    # 更新邻居节点
                    neighbor.parent = current_node
                    neighbor.g = temp_g
                    neighbor.h = self._calculate_h(neighbor, end_node)
                    neighbor.f = neighbor.g + neighbor.h
                    
                    if neighbor not in open_list:
                        heapq.heappush(open_list, neighbor)
        
        return []  # 无法找到路径
    
    def _reconstruct_path(self, end_node):
        """回溯构建路径"""
        path = []
        current = end_node
        
        while current:
            path.append((current.world_x, current.world_z))
            current = current.parent
        
        return path[::-1]  # 反转路径，从起点到终点
```

```python
def update_path(self):
    """更新路径"""
    self.path = self.pathfinder.find_path(
        (self.position.x, self.position.y, self.position.z),
        (player.position.x, player.position.y, player.position.z)
    )
    self.current_path_index = 0

def follow_path(self):
    """跟随路径移动"""
    if not self.path or self.current_path_index >= len(self.path):
        return
    
    # 获取当前目标点
    target_x, target_z = self.path[self.current_path_index]
    target_pos = Vec3(target_x, self.position.y, target_z)
    
    # 计算移动方向
    move_direction = target_pos - self.position
    move_direction.y = 0
    move_direction = move_direction.normalized()
    
    # 移动到目标点
    move_distance = 5 * time.dt
    if distance(self.position, target_pos) > move_distance:
        self.position += move_direction * move_distance
    else:
        # 到达当前目标点，移动到下一个点
        self.position = target_pos
        self.current_path_index += 1
```

## 6. 性能优化

### 6.1 算法优化
- 使用优先队列（heapq）提高开放列表的访问效率
- 限制路径搜索深度，避免无限循环
- 缓存最近计算的路径，减少重复计算

### 6.2 实现优化
- 降低路径更新频率（默认0.5秒）
- 仅当玩家移动超过一定距离时才更新路径
- 对复杂迷宫进行预计算，生成导航网格

### 6.3 内存优化
- 每次路径计算后重置节点状态
- 避免频繁创建新对象
- 合理设置路径点数量上限
