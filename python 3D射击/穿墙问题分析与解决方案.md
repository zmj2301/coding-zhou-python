# 敌人穿墙问题分析与解决方案

## 1. 问题描述
敌人可以直接穿过迷宫墙壁，不受碰撞系统限制，这破坏了游戏的物理规则和可玩性。

## 2. 根本原因分析

### 2.1 现有代码结构
- **墙壁创建**：第72-80行创建迷宫墙壁，每个墙壁都设置了`collider='box'`
- **敌人创建**：第82-121行定义Enemy类，敌人也设置了`collider='box'`
- **敌人移动逻辑**：第100-101行
  ```python
  if distance(self.position,player.position) > 5:
      self.position = self.position + self.forward * 5 * time.dt
  ```

### 2.2 问题核心
敌人的移动是**直接修改position属性**，而不是使用Ursina的碰撞检测系统来处理移动。

这种直接赋值方式会绕过所有碰撞检测，导致敌人直接穿过墙壁。

## 3. Ursina碰撞系统工作原理

Ursina的碰撞检测需要以下条件：
1. 两个实体都必须有`collider`属性
2. 移动必须通过**碰撞感知的方法**进行，而不是直接修改position
3. 推荐使用`move_towards`、`lerp`或物理引擎来处理移动

## 4. 解决方案

### 4.1 方案一：使用`move_towards`配合碰撞检测
```python
def update(self):
    self.look_at_2d(player.position,'y')
    if distance(self.position,player.position) > 5:
        # 使用move_towards代替直接赋值
        move_direction = player.position - self.position
        move_direction.y = 0  # 只在水平面上移动
        move_direction = move_direction.normalized()
        
        # 检查碰撞
        hit_info = raycast(self.position, move_direction, distance=5*time.dt, ignore=(self,))
        if not hit_info.hit:
            self.position += move_direction * 5 * time.dt
```

### 4.2 方案二：使用`lerp`配合碰撞检测
```python
def update(self):
    self.look_at_2d(player.position,'y')
    if distance(self.position,player.position) > 5:
        target_position = player.position
        
        # 计算移动方向
        move_direction = target_position - self.position
        move_direction.y = 0
        move_direction = move_direction.normalized()
        
        # 尝试移动
        new_position = self.position + move_direction * 5 * time.dt
        
        # 检查新位置是否与墙壁碰撞
        hit_info = boxcast(self.position, self.scale, direction=move_direction, distance=5*time.dt, ignore=(self,))
        if not hit_info.hit:
            self.position = new_position
```

### 4.3 方案三：使用Ursina的物理引擎
```python
# 在Enemy类初始化中添加
self.enable_physics = True
self.velocity = Vec3(0, 0, 0)

# 在update方法中
self.look_at_2d(player.position,'y')
if distance(self.position,player.position) > 5:
    self.velocity = self.forward * 5
else:
    self.velocity = Vec3(0, 0, 0)
```

## 5. 推荐实施方案

### 5.1 选择理由
- 方案一（`move_towards`）实现简单，适合当前游戏规模
- 性能开销小，不会影响游戏流畅度
- 碰撞检测准确，能有效防止穿墙

### 5.2 具体实现步骤
1. 修改Enemy类的update方法
2. 替换直接position赋值为带碰撞检测的移动
3. 测试不同场景下的碰撞效果
4. 调整移动速度和碰撞检测参数

- 游戏物理规则更加真实
- 提高游戏的可玩性和挑战性

## 9. 结论

敌人穿墙问题是由于移动逻辑设计不当导致的，通过使用Ursina的碰撞检测系统配合合适的移动方法，可以有效解决这个问题。推荐使用`move_towards`配合`raycast`的方案，实现简单且效果良好。

这个修复将显著提高游戏的真实感和可玩性，让玩家体验到更加合理的敌人AI行为。
